# 📧 Email Triage Environment (OpenEnv)

## 🚀 Overview

The **Email Triage Environment** simulates a real-world customer support workflow where an AI agent must **classify and respond to incoming emails**.

This environment is designed to evaluate an agent’s ability to:

* Understand user intent
* Categorize emails correctly
* Generate helpful and professional responses

It follows the **OpenEnv standard** (`step() / reset() / state()`) and provides **graded tasks with continuous reward signals**.

---

## 🎯 Motivation

Customer support teams handle thousands of emails daily. Automating email triage can:

* Reduce response time
* Improve customer satisfaction
* Scale support operations

This environment enables training and evaluation of AI agents for **real-world support automation tasks**.

---

## 🧠 Environment Design

### Observation Space

```json
{
  "email_text": "string"
}
```

Represents the incoming customer email.

---

### Action Space

```json
{
  "category": "string (refund | complaint | angry)",
  "reply": "string"
}
```

* **category** → classification of the email
* **reply** → response generated by the agent

---

### Reward Function (0.0 → 1.0)

The reward is **dense and interpretable**, based on:

| Component         | Weight |
| ----------------- | ------ |
| Correct category  | 0.5    |
| Keyword relevance | 0.3    |
| Response quality  | 0.2    |

#### Reward Signals:

* ✅ Correct classification → +0.5
* ✅ Relevant keywords → up to +0.3
* ✅ Polite & detailed reply → up to +0.2
* ❌ Invalid category → penalty
* ❌ Empty reply → penalty

---

## 📋 Tasks

The environment includes **3 tasks with increasing difficulty**:

### 🟢 Easy — Refund Request

* Example: *"I want a refund for my recent purchase"*
* Goal: Identify refund intent and guide user

---

### 🟡 Medium — Complaint Handling

* Example: *"This product quality is very bad"*
* Goal: Apologize and propose resolution

---

### 🔴 Hard — Angry Customer

* Example: *"I am extremely angry about your service"*
* Goal: De-escalate and respond empathetically

---

## 🔄 API Endpoints

| Endpoint    | Description                   |
| ----------- | ----------------------------- |
| `/reset`    | Start a new task              |
| `/step`     | Take an action                |
| `/state`    | Get current environment state |
| `/tasks`    | List tasks + schema           |
| `/baseline` | Run baseline agent            |
| `/grader`   | Return grading info           |
| `/test`     | Health test                   |
| `/health`   | API health check              |

## ⚙️ Example Usage

### Reset Environment

```
GET /reset
```

Response:

```json
{
  "email_text": "I want a refund for my recent purchase"
}
```

---

### Take Action

```
POST /step
```

```json
{
  "category": "refund",
  "reply": "You can request a refund through our refund policy process."
}
```

Response:

```json
{
  "observation": {...},
  "reward": {"score": 0.9},
  "done": true,
  "info": {}
}
```
